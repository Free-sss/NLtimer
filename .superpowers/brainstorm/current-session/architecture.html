设计总览 — NLtimer 主题系统架构

方案 A：独立 core:designsystem 模块 · 完整复刻 Momentum 主题能力

🏗️ 模块分层

📱 app 模块
NLtimerApplication · NLtimerTheme.kt · NLtimerScaffold
SettingsScreen（主题设置 UI）· 各 feature 页面
⚡ 职责：组装 DI、提供入口 Composable、主题设置页面 v2
⬆️ depends on ⬆️
🎨 core:designsystem 模块
Theme.kt · Fonts.kt · PaletteStyle.kt · AppTheme.kt
NLtimerTheme.kt · provideTypography.kt
listItemExt.kt · 通用组件样式
⚡ 职责：主题数据模型、主题渲染引擎、组件样式规范 —— 被所有模块依赖
⬆️ depends on ⬆️
💾 core:data 模块
SettingsPrefs.kt（接口）· SettingsPrefsImpl.kt
DatastoreFactory.kt · ThemePreferencesKeys
⚡ 职责：主题配置的 DataStore 持久化，提供 Flow 流式读取和更新

🔄 数据流

👤 用户操作 (SettingsScreen)
  → SettingsAction (sealed interface)
  → SettingsViewModel (Hilt)
  → SettingsPrefsImpl.updateXxx() → DataStore
  → SettingsPrefs.getXxxFlow() → StateFlow<Theme>
  → MainAppViewModel.state (MainAppState.theme)
  → 🎨 NLtimerTheme(theme) → DynamicMaterialTheme
  → 所有子 Composable 自动继承主题
