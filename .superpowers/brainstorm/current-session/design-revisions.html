② 修正：去掉 MainAppViewModel 主题瓶颈

修正后的数据流

MainActivity.setContent {
  val theme by settingsPrefs.getThemeFlow()
    .collectAsStateWithLifecycle(Theme())
  NLtimerTheme(theme = theme) {
    NLtimerApp() // 所有子组件自动继承
  } }

// SettingsViewModel 只处理设置页 UI 逻辑
// 不持有全局 Theme 状态，调用 settingsPrefs.updateTheme()
// DataStore 写入 → Flow 自动发射 → setContent 层重组

原来的问题

❌ MainAppViewModel 必须全局存活
❌ 子组件需向上依赖 ViewModel 或逐层传参
❌ ViewModel 和 Compose CompositionLocal 双轨维护 Theme 状态
💡 设计原则：Theme 属于 Compose 层级的状态（通过 CompositionLocal 分发），不属于 ViewModel 的职责。SettingsViewModel 只有"写入"操作，没有"持有"操作。顶层 setContent 通过 collectAsStateWithLifecycle 直接订阅 DataStore Flow。

④ 新增：主题过渡防闪烁策略

NLtimerTheme(theme: Theme, content: @Composable () → Unit) {
  val animatedScheme = animateColorScheme(
    targetScheme = dynamicScheme,
    animationSpec = tween(300ms)
  )
  DynamicMaterialTheme(scheme = animatedScheme, ...) {
    AnimatedContent(targetState = theme, fadeIn + fadeOut) {
      content()
    }
  }
}
✅ 色彩过渡：animateColorScheme 让颜色平滑渐变而非突变
✅ 内容过渡：AnimatedContent + fadeIn/fadeOut 消除视觉闪烁
✅ 稳定 key：Theme data class 直接作为 key，Compose 正确识别变更

📋 修订后的最终数据流
// === 写入路径（设置页触发）===
SettingsScreen → SettingsAction → SettingsViewModel
  → settingsPrefs.updateTheme(newTheme) → DataStore

// === 读取路径（Compose 声明式）===
DataStore → getThemeFlow(): Flow<Theme>
  → MainActivity.setContent 层 collectAsStateWithLifecycle
  → 🎨 NLtimerTheme(theme, animatedScheme)
  → CompositionLocal 自动分发到所有子组件

MaterialTheme.colorScheme / MaterialTheme.typography
