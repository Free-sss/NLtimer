
## 行为时间记录系统 · 最终逻辑规格 v 2.1

### 一、核心数据模型

**行为对象**
- 独立实体，包含：唯一 ID、活动名称（从预设列表选择）、标签（可选）、开始类型、开始时间、结束时间（可为空）。
- **一次性**：行为创建后不可重用。

**三种行为容器**
- **目标队列（FIFO）**：类型为“目标”的预定义行为，待自动开始。
- **当前槽位（唯一）**：系统中唯一正在计时的行为，类型为“当前”。
- **完成队列**：所有已结束行为，类型为“完成”。

**开始类型**
记录行为是如何开始的，用于详情展示：
- `target` – 由目标队列自动开始
- `current` – 通过【当前】弹窗手动开始
- `complete` – 通过【完成】弹窗直接补录

---

### 二、行为生命周期

1. **标准路径**：`目标` → `当前` → `完成`  
   目标在队列中等待 → 被自动开始（变成当前） → 计时结束（变成完成）  
   *此路径完全自动，由系统触发。*

2. **手动当前路径**：`当前` → `完成`  
   用户通过【当前】弹窗直接创建并开始一个行为（不依赖目标） → 计时结束  
   **准入条件：仅当目标队列为空时，【当前】选项可用。**  
   若目标队列非空，该选项禁用，用户只能选择【目标】或【完成】。

3. **直接完成路径**：`完成`  
   用户通过【完成】弹窗直接补录一条已完成记录（无当前阶段）

---

### 三、【目标】模块

- 弹窗无时间选择器，仅提供活动选择（预设列表）、标签选择（可选）。
- 提交后生成一条类型为“目标”的行为，追加到**目标队列尾部**。
- 提交后检查：若**当前槽位为空**且目标队列非空，立即**自动开始**队头目标（开始时间 = 系统当前时间，类型 = `target`）。

---

### 四、【当前】模块（手动计时）

**核心约束**
- 系统最多同时有一个当前行为。
- 弹窗**不展示任何行为数据**（不列目标、不选目标），仅提供增加功能。
- **准入控制**：仅当目标队列为空时，该模块可用；否则按钮/入口为禁用态。

**场景一：当前无活动且目标队列为空，用户手动开始**
- 弹窗提供：活动名称（从预设列表选择）、标签、开始时间选择器。
- 开始时间可选过去时间，但必须不与任何已有完成记录或当前行为区间重叠。
- 提交后，新行为直接占据当前槽位，开始计时，开始类型 = `current`。

**场景二：当前有活动且目标队列为空，用户手动切换（开始新行为）**
- 打开弹窗瞬间，记录系统时间 `T_cut`。
- 弹窗提供：活动名称、标签、开始时间选择器、**“上尾”勾选框**。
  - 若勾选“上尾”（默认勾选），开始时间自动设为 `T_cut`。
  - 若取消勾选，可手动选择开始时间，但**不得早于 `T_cut` **。
- 提交后执行：
  1. 旧当前行为结束时间设为 `T_cut`，类型变为“完成”，移入完成队列。
  2. 新行为以所选开始时间（≥ `T_cut`）开始，占据当前槽位，开始类型 = `current`。
- **注意**：此切换不触发自动开始（因目标队列为空，且用户已手动衔接）。

**场景三：当前无活动且目标队列为空**，同场景一。

**场景四：主页“完成当前计时”按钮**
- 点击瞬间，以系统当前时间作为结束时间。
- 当前行为类型变为“完成”，移入完成队列。
- 随后检查：若目标队列非空，立即**自动开始**队头目标（开始时间 = 当前时间，类型 = `target`）；若目标队列为空，系统进入空闲。

---

### 五、【完成】模块（事后补录）

- 完全独立，不受当前计时影响，也不影响当前计时。
- 弹窗提供：活动名称、标签、开始时间、结束时间（均必填）。
- 提交后创建一条类型为“完成”的行为，开始类型 = `complete`，直接加入完成队列。
- 补录时间段必须完全位于时间轴空闲区域，不与任何记录重叠（含进行中的当前行为区间 `[当前开始, +∞)`）。
- **不触发**自动开始。
- **不受目标队列状态影响，始终可用。**

---

### 六、自动开始规则（目标→当前）

**触发时机（仅在三处）**
1. 主页“完成当前计时”后，当前槽位变空时。
2. 在【目标】弹窗添加新目标后，若当前无活动。
3. 应用启动时，若当前无活动且目标队列非空。

**执行逻辑**
- 从目标队列**头部（FIFO）**取出目标。
- 以**系统当前时间**为开始时间，将其类型变为“当前”，移入当前槽位，开始类型标记为 `target`。
- 若目标队列为空，保持空闲，不做任何操作。

---

### 七、时间轴重叠约束

- 所有时间区间采用 ** `[开始时间, 结束时间)` ** 左闭右开。
- 进行中的当前行为视为 ** `[开始时间, +∞)` **。
- 新增任何记录（手动当前或补录完成）必须不与已有区间重叠。
- 切换当前行为时，旧行为被 `T_cut` 封闭为 `[旧开始, T_cut)`，新行为从 `≥ T_cut` 开始，保证无冲突。

---

### 八、更新后的边界行为速查表

| 场景 | 系统行为 |
|------|----------|
| 目标队列**不为空**，用户点击【当前】 | **入口禁用**，不可操作 |
| 目标队列为空，当前无活动，点【当前】手动开始 | 正常创建当前行为（无目标关联） |
| 目标队列为空，当前有活动，点【当前】手动切换 | 以 `T_cut` 结束旧行为，开始新行为 |
| 点主页“完成当前计时”（无论目标队列是否空） | 结束当前；若目标队列非空则立即自动开始队头目标，若空则空闲 |
| 补录【完成】时间段与任何已有记录重叠 | 拒绝，提示调整 |
| 手动当前选时早于已有完成记录 | 拒绝 |
| 添加目标时当前空闲 | 立即自动开始该目标 |
| 添加目标时当前忙碌 | 目标仅入队，不打扰当前计时 |
| 任何时刻使用【完成】补录 | 直接写入，不干扰当前行为 |
