# AI 上下文文档维护指南

> 修改代码后，按此流程同步更新 `docs/ai-context/` 文档，确保后续 AI agent 获取的信息始终准确。

## 触发条件

| 修改类型 | 需要更新的文档 |
|---------|--------------|
| 新增/删除/重命名模块 | 00-index（目录树）、01-architecture（依赖/导航）、04-module-detail |
| 新增/修改数据表或字段 | 02-data-model |
| 新增/升级依赖 | 03-tech-stack |
| 新增/修改枚举、配置类 | 02-data-model（如果是模型）、04-module-detail（如果是 UI） |
| 新增/修改 Route/Screen/ViewModel | 04-module-detail（对应模块表格） |
| 新增/修改数据库迁移 | 02-data-model（迁移链） |
| 修改命名/架构/代码规范 | 05-conventions |
| 修改导航路由 | 01-architecture（导航结构） |

## 更新原则

1. **只改受影响的部分** — 不重写整个文档，只更新变化的行/表项
2. **保持表格格式** — 新增条目追加到表格末尾，保持字段对齐
3. **保持树结构** — 目录树中新增模块按字母序插入
4. **不改编号** — 文档编号 00~05 固定，不因内容变化重新编号

## 快速检查清单

修改代码后，依次确认：

- [ ] 00-index：目录树是否需要新增/删除条目？
- [ ] 01-architecture：模块依赖方向、导航路由、DI 绑定是否变化？
- [ ] 02-data-model：实体字段、关联关系、迁移版本是否变化？
- [ ] 03-tech-stack：依赖版本号是否变化？
- [ ] 04-module-detail：模块内文件列表是否变化？
- [ ] 05-conventions：命名/模式约定是否变化？

## 不需要更新的情况

- 纯 UI 样式调整（颜色值、间距、动画参数）
- Bug 修复（不改变接口/结构）
- 资源文件修改（strings.xml、字体、图片）
- 注释修改
- 测试代码修改
